# リリース前チェックの手順

## 1. 機械的チェック(すべて必須)

```bash
npm run lint
npm run typecheck
npm test
npm run build
```

一つでも失敗したら不合格。失敗内容を原文のまま報告し、修正は開発部(engineer)に委任する。

## 2. コンセプト適合チェック

変更差分(`git diff`)を見て、以下に反していないか確認する:

- 採点・点数・ランキング・正解判定を出していないか
- 否定的フィードバック(失敗、声が小さい、下手など)を表示していないか
- 音声データを端末外に送信 or 保存するコードが入っていないか
- 5秒未満の録音を「責める」表現になっていないか

## 3. 報告形式

- 機械的チェック: 各コマンドの合格/不合格
- コンセプト適合: 問題なし / 要修正(該当箇所を file:line で指摘)
